# 관리자 설정

## 기본 정보

- Route: `/admin-settings`
- 주요 파일: `src/pages/AdminSettingsPage.tsx`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 가격, 회사, 권한, 템플릿, 연동 설정 등 관리 기능으로 진입하는 허브다.

## UI 구조

- 상단: 관리자 설정 제목, 홈/뒤로 액션
- 섹션: 가격/원판, 회사/문서, 직원/권한, 외부 연동, 시스템 관리
- 카드: 기능명, 설명, 접근 권한, 이동 버튼
- 제한 상태: 권한이 없을 때 안내

## 톤앤매너

- 설정 허브는 메뉴판처럼 단순하고 예측 가능해야 한다.
- 카드 제목, 아이콘, 설명 길이를 통일한다.
- 위험한 설정은 일반 설정보다 시각적으로 구분한다.

## 자주 수정하는 요소

- 설정 카드 그룹
- 관리자 전용 배지
- 외부 연동 설명
- 접근 제한 문구

## 연결 데이터와 주의점

- 연결 데이터: 페이지 접근 권한, 회사 설정, 각 관리 페이지
- 권한: 관리자/중간관리자 중심
- 변경 위험: 메뉴 노출은 보안 경계가 아니므로 실제 권한 guard와 RLS를 유지해야 한다.

## 정보 위계

- 1순위: 가격/원판/가공처럼 업무 영향이 큰 설정
- 2순위: 회사 정보, 견적 문구, 권한 설정
- 3순위: 외부 연동, 보조 관리 도구
- 낮은 강조: 거의 사용하지 않는 실험성 설정

## 상세 설계 기준

- 그룹 구성: 카드 그룹은 업무 흐름 기준으로 묶는다. 예: 가격 관리, 운영 관리, 직원 관리, 외부 연동.
- 카드: 모든 설정 카드는 아이콘, 제목, 한 줄 설명, 이동 액션을 같은 구조로 둔다.
- 위험 설정: 권한/가격/외부 API 설정은 일반 카드보다 보조 경고 문구를 둔다.
- 접근 제한: 권한이 없으면 숨김보다 "접근 불가"를 명확히 보여주는 경우가 안전한지 판단한다.
- 모바일: 설정 카드는 1열로 쌓고 설명 문구는 두 줄 이내로 제한한다.

## 검수 시나리오

- 일반 직원으로 접근 시 권한 제한 화면이 나온다.
- 관리자 카드 클릭이 실제 관리 route로 이동한다.
- 중복 설정 카드가 없고 같은 기능이 두 이름으로 보이지 않는다.
- 새 관리 기능을 추가할 때 README와 해당 페이지 spec 링크가 함께 갱신된다.
