# 회사 설정

## 기본 정보

- Route: `/company-settings`
- 주요 파일: `src/pages/CompanySettingsPage.tsx`, `src/components/company/*`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 회사 정보, 견적 기본 문구, 휴일, 권한 정책을 설정한다.

## UI 구조

- 상단: 회사 설정 제목, 탭/저장 액션
- 섹션: 회사 정보, 견적 기본 문구, 휴일, 페이지 권한, 기능 권한
- 입력폼: 회사명, 사업자 정보, 계좌, 연락처, 문구
- 관리 표: 권한/휴일 목록

## 톤앤매너

- 회사 정보는 견적서 출력에 영향을 주므로 안정적인 폼 레이아웃을 유지한다.
- 권한 설정은 표 형태로 명확히 보여준다.
- 저장 성공/실패 메시지를 짧게 제공한다.

## 자주 수정하는 요소

- 견적 기본 문구
- 회사 정보 폼 라벨
- 권한 테이블
- 휴일 관리 카드

## 연결 데이터와 주의점

- 연결 데이터: 회사 설정, 견적 템플릿, `page_role_access`
- 권한: 관리자/중간관리자 전용
- 변경 위험: 회사 정보는 고객용 견적서/PDF에 그대로 노출될 수 있다.

## 정보 위계

- 1순위: 고객용 견적서에 노출되는 회사명, 연락처, 계좌, 기본 문구
- 2순위: 페이지/기능 권한, 휴일 정책
- 3순위: 내부 운영 메모, 보조 설정
- 낮은 강조: 변경 빈도가 낮은 레거시 필드

## 상세 설계 기준

- 견적 문구: 고객에게 직접 노출되는 문구는 미리보기와 함께 편집한다.
- 권한 관리: 역할별 접근은 표 형태로 보이고 route 이름보다 화면 이름을 우선 표기한다.
- 회사 정보: 입력 필드는 실제 견적서 표시 순서와 유사하게 배치한다.
- 휴일: 캘린더/목록 양쪽에서 같은 휴일명이 보이도록 한다.
- 모바일: 긴 문구 편집 영역은 충분한 높이를 확보하고 저장 버튼은 섹션별로 명확히 둔다.

## 검수 시나리오

- 회사 정보 변경 후 견적서 상세/PDF 표시가 의도와 맞는지 확인한다.
- 권한 변경 후 대상 계정으로 실제 route 접근을 확인한다.
- 휴일 추가 후 홈/근태/연차 화면의 날짜 계산 영향이 없는지 확인한다.
- 저장 실패 시 어떤 섹션의 저장이 실패했는지 알 수 있다.
