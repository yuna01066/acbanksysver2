# 단가 관리

## 기본 정보

- Route: `/price-management`
- 주요 파일: `src/pages/PriceManagement.tsx`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 판재 단가와 기준 가격 정보를 관리한다.

## UI 구조

- 상단: 단가 관리 제목, 저장/버전 관련 액션
- 본문: 단가표, 기준일, 버전, 금액 입력
- 보조: 변경 안내, 적용 범위, 검증 메시지
- 빈 상태: 단가 데이터 없음 또는 로딩 실패

## 톤앤매너

- 금액 입력 화면은 시각적 안정성과 숫자 가독성이 중요하다.
- 경고 문구는 짧고 구체적으로 유지한다.
- 단가표 버전명은 `YYYY-MM 단가표`처럼 거래처명 없이 표시한다.

## 자주 수정하는 요소

- 단가표 헤더
- 기준일/버전 배지
- 저장 버튼과 검증 메시지
- 단가 없음 fallback

## 연결 데이터와 주의점

- 연결 데이터: 단가 관련 Supabase 테이블, 계산 snapshot
- 권한: 관리자/중간관리자 전용
- 변경 위험: 단가 변경은 신규 계산에만 반영하고 기존 저장 견적 금액은 보호한다.

## 정보 위계

- 1순위: 활성 단가표, 기준월, 주요 규격별 금액
- 2순위: 버전 상태, 적용 시작일, 변경 저장 상태
- 3순위: 내부 메모, 공급처 원본 정보
- 낮은 강조: 과거 비활성 버전

## 상세 설계 기준

- 단가표명: 모든 사용자-facing 표시는 `YYYY-MM 단가표`로 통일한다.
- 입력 표: 행은 두께, 열은 사이즈처럼 원본 단가표 구조에 가깝게 유지한다.
- 변경 안내: "기존 견적에는 자동 반영되지 않음"을 저장 버튼 주변에 짧게 표시한다.
- 숫자 검증: 빈 값, 0, 음수, 쉼표 포함 입력을 구분해 안내한다.
- 모바일: 단가 편집은 데스크톱 최적화로 두되 모바일에서는 조회 중심으로 무리 없이 보이게 한다.

## 검수 시나리오

- 활성 단가표가 계산기 계산 근거에 같은 이름으로 표시된다.
- 신규 단가 저장 후 새 계산에는 반영되고 기존 견적 상세는 유지된다.
- 날짜 없는 단가표는 `기준 단가표` fallback을 사용한다.
- 저장 실패 시 변경된 셀을 사용자가 다시 찾을 수 있다.
