# 원판/색상/사이즈 관리

## 기본 정보

- Route: `/panel-management`
- 주요 파일: `src/pages/PanelManagementPage.tsx`, `src/components/panel-management/*`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 판재 제품, 컬러, 사이즈, 두께, 추가금과 생산 가능 조합을 관리한다.

## UI 구조

- 상단: 원판 관리 제목, 탭/관리 액션
- 관리 영역: 제품 선택, 컬러 관리, 사이즈 관리, 옵션/추가금 관리
- 목록/폼: 생산 가능 여부, 단가, 조색비, 양단면 추가금
- 피드백: 저장 성공/오류, 단가 미등록 안내

## 톤앤매너

- 관리자 화면이므로 장식보다 입력 정확도와 검증 메시지를 우선한다.
- 컬러/사이즈/두께 조합은 표나 그룹으로 정리해 실수 가능성을 낮춘다.
- 거래처명은 외부 노출 화면에 나오지 않게 관리용 정보로만 둔다.

## 자주 수정하는 요소

- 컬러 속성/배지
- 사이즈별 단가 입력 표
- 생산 불가 조합 표시
- 양단면/사틴/아스텔/조색비 문구

## 연결 데이터와 주의점

- 연결 데이터: `panel_masters`, `panel_sizes`, `panel_option_surcharges`, `panel_pricing_versions`
- 권한: 관리자/중간관리자 전용
- 변경 위험: 단가 변경은 기존 저장 견적에 자동 반영되지 않아야 한다.

## 정보 위계

- 1순위: 현재 제품/컬러/사이즈 조합, 생산 가능 여부, 단가
- 2순위: 사틴/아스텔/양단면/조색비 같은 추가금
- 3순위: 단가표 버전, 공급처 내부 메모
- 낮은 강조: 오래된 비활성 조합

## 상세 설계 기준

- 조합 관리: 제품 -> 컬러 -> 두께 -> 사이즈 -> 옵션 순으로 선택하게 해 현재 편집 맥락을 잃지 않게 한다.
- 단가 입력: 숫자 입력은 우측 정렬하고 빈 값은 생산 불가로 해석된다는 안내를 둔다.
- 색상 속성: CLEAR, 사틴/아스텔, 브라이트/흰색 안료 추가 같은 속성을 배지로 분리한다.
- 버전 표시: 사용자 화면에는 `YYYY-MM 단가표`, 관리자 화면에는 필요 시 내부 공급처 정보를 보조로만 둔다.
- 모바일: 큰 단가표는 가로 스크롤을 허용하되 행/열 헤더가 무엇인지 잃지 않게 한다.

## 검수 시나리오

- 빈 금액 칸 조합이 계산기에서 생산 불가로 처리된다.
- 양단면 추가금이 기존 두께별 추가금과 중복 표시되지 않는다.
- 사틴/아스텔 추가금이 유광 기준 금액에 더해지는 구조로 설명된다.
- 단가표 버전을 변경해도 기존 저장 견적 상세 금액은 변하지 않는다.
