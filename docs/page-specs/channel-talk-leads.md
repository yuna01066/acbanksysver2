# 채널톡 문의 분석함

## 기본 정보

- Route: `/channel-talk-leads`
- 주요 파일: `src/pages/ChannelTalkLeadsPage.tsx`, `src/components/ChannelTalkInquiryCard.tsx`
- 주요 사용자: 전체 직원, 관리자, 중간관리자
- 화면 목적: 채널톡에서 들어온 문의와 AI 분석 결과를 확인하고 내부 응대 참고 자료로 사용한다.

## UI 구조

- 상단: 채널톡 문의 분석함 제목, 필터/검색
- 목록: 고객/회사, 문의 분류, 신뢰도, 누락 정보, 상태, 생성 시간
- 상세: AI 분석 요약, 예상 답변, 확인 체크리스트, 상담원 메모, 원본 payload 요약
- 액션: 상태 변경, 담당자 지정, 메모 수정, 복사, 샘플칩 재고 확인

## 톤앤매너

- 문의 분류 배지는 `샘플칩`, `제작`, `검토 필요`가 즉시 보이게 한다.
- 예상 답변은 내부 참고용임을 분명히 하고 고객 자동 발송처럼 보이지 않게 한다.
- 일반 직원은 읽기/복사 중심, 관리자 액션은 분리한다.

## 자주 수정하는 요소

- 문의 카드 요약 정보
- 분류/신뢰도/누락 정보 배지
- 예상 답변 섹션
- 읽기 전용/관리자 액션 구분

## 연결 데이터와 주의점

- 연결 데이터: `channel_talk_quote_leads`, `notifications`
- 권한: 전체 직원 SELECT, 관리자/중간관리자 UPDATE
- 변경 위험: 고객에게 자동 답변을 보내는 기능은 현재 범위가 아니다.

## 정보 위계

- 1순위: 문의 분류, 고객/회사, 요약, 생성 시간
- 2순위: AI 신뢰도, 누락 정보 수, 추천 답변
- 3순위: 상담원 메모, 원본 payload 요약, 첨부 키
- 낮은 강조: 내부 분석 로그, webhook 원본 세부값

## 상세 설계 기준

- 분류 배지: `샘플칩`, `제작`, `혼합`, `검토 필요`는 목록과 상세에서 같은 라벨/색을 사용한다.
- 홈 카드: 최근 3-5건을 읽기 전용으로 보여주고 상태 변경/견적 생성 액션은 노출하지 않는다.
- 상세: 예상 답변은 복사 버튼을 제공하되 "고객 자동 발송 아님"을 명확히 한다.
- 권한: 일반 직원은 읽기/복사만 가능하고 관리자/중간관리자에게만 상태/담당자/메모 액션을 연다.
- 모바일: 목록과 상세를 한 화면에 병치하지 말고 선택 후 상세 진입 흐름을 우선한다.

## 검수 시나리오

- 텍스트만 있는 샘플칩 문의가 샘플칩 배지와 예상 답변을 보여준다.
- 제작 문의는 사이즈/수량/소재/납기 누락 항목을 체크리스트로 보여준다.
- 일반 직원 계정에서 상태 변경 버튼이 보이지 않는다.
- `converted`와 `closed` 문의가 홈 카드에 노출되지 않는다.
