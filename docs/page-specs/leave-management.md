# 연차 관리

## 기본 정보

- Route: `/leave-management`
- 주요 파일: `src/pages/LeaveManagementPage.tsx`, `src/components/leave/*`
- 주요 사용자: 전체 직원, 관리자
- 화면 목적: 연차 신청, 승인, 잔여일수, 정책과 사용 내역을 관리한다.

## UI 구조

- 상단: 연차 관리 제목, 신청 액션
- 개인 영역: 잔여 연차, 사용/예정, 신청 내역
- 관리자 영역: 승인 대기, 정책 설정, 직원별 현황
- 캘린더/테이블: 휴가 일정과 기록

## 톤앤매너

- 잔여일수와 승인 상태가 가장 먼저 보여야 한다.
- 승인/반려 버튼은 명확히 구분한다.
- 정책 설정은 일반 신청 화면과 분리해 복잡도를 낮춘다.

## 자주 수정하는 요소

- 연차 요약 카드
- 신청 폼 라벨
- 승인 대기 목록
- 정책/휴가 유형 카드

## 연결 데이터와 주의점

- 연결 데이터: 연차 요청, 연차 정책, 사용자 프로필, 회사 휴일
- 권한: 신청자는 본인 요청, 관리자는 전체 승인/정책
- 변경 위험: 잔여일수 계산 로직과 UI 표시를 혼동하지 않는다.

## 정보 위계

- 1순위: 잔여 연차, 승인 대기/진행 중 요청, 신청 버튼
- 2순위: 사용 예정, 사용 완료, 캘린더 일정
- 3순위: 정책 설정, 휴가 유형, 상세 사용 내역
- 낮은 강조: 오래된 만료/조정 로그

## 상세 설계 기준

- 개인 화면: 잔여일수와 신청 액션을 상단에 두고 신청 내역은 상태별로 구분한다.
- 관리자 화면: 승인 대기 목록을 가장 먼저 보여주고 정책 설정은 별도 탭/섹션으로 분리한다.
- 상태 배지: 신청, 승인, 반려, 취소, 사용 완료를 같은 배지 체계로 표시한다.
- 캘린더: 회사 휴일과 개인 휴가가 시각적으로 구분되어야 한다.
- 모바일: 신청 폼은 단계형으로 단순화하고 날짜 선택이 터치하기 쉬워야 한다.

## 검수 시나리오

- 일반 직원으로 신청 후 승인 대기 상태가 개인 목록에 표시된다.
- 관리자가 승인/반려하면 개인 잔여일수와 상태가 갱신된다.
- 반차/특수 휴가 유형이 표기와 계산에서 일치한다.
- 회사 휴일이 휴가 일수 계산에 잘못 포함되지 않는다.
