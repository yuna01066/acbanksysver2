# 근태 관리

## 기본 정보

- Route: `/attendance`
- 주요 파일: `src/pages/AttendancePage.tsx`, `src/components/QuickAttendanceButton.tsx`
- 주요 사용자: 전체 직원, 관리자
- 화면 목적: 출퇴근 체크, 근무 기록, 출근 상태를 관리한다.

## UI 구조

- 상단: 근태 관리 제목, 출근/퇴근 액션
- 개인 영역: 오늘 상태, 출퇴근 시간, 연속 기록, 근무 진행률
- 목록/캘린더: 날짜별 근태 기록
- 관리자 영역: 직원별 현황, 수정/승인 액션

## 톤앤매너

- 출근/퇴근 버튼은 명확하고 실수 없이 누를 수 있어야 한다.
- 성취감 요소는 작게 유지하고 업무 화면의 안정성을 해치지 않는다.
- 시간과 상태 표기는 같은 형식을 따른다.

## 자주 수정하는 요소

- 출근/퇴근 버튼 문구
- 연속 출근 기록 표시
- 근무 진행률/점심시간 아이콘 효과
- 관리자 현황 테이블

## 연결 데이터와 주의점

- 연결 데이터: 근태 기록, 프로필, 회사 휴일/근무 정책
- 권한: 개인 기록과 관리자 전체 조회 분리
- 변경 위험: UI 표시 변경이 실제 출퇴근 기록 생성/수정 로직에 영향을 주지 않도록 한다.

## 정보 위계

- 1순위: 오늘 출근 상태, 출근/퇴근 버튼, 현재 시간
- 2순위: 근무 진행률, 출퇴근 기록, 연속 정상 출근
- 3순위: 월간 기록, 관리자 현황, 수정 요청
- 낮은 강조: 장식성 효과, 보조 설명

## 상세 설계 기준

- 오늘 상태: 체크 전/근무 중/퇴근 완료 상태를 버튼 문구와 색으로 명확히 구분한다.
- 이스터에그: 점심시간 아이콘, 연속 출근 기록은 작게 보여주고 핵심 버튼보다 튀지 않게 한다.
- 기록 표: 날짜, 출근, 퇴근, 상태, 메모 순서로 일관되게 표시한다.
- 관리자: 직원별 수정/승인 액션은 일반 직원 UI와 분리한다.
- 모바일: 출퇴근 버튼은 엄지로 누르기 쉬운 크기를 유지하고 기록 표는 카드형으로 전환한다.

## 검수 시나리오

- 출근 전, 근무 중, 퇴근 후 상태별 버튼 문구가 맞는지 확인한다.
- 점심시간대 효과가 표시되어도 진행률/시간 텍스트를 가리지 않는다.
- 주 5일 연속 정상 출근 문구가 조건에 맞을 때만 표시된다.
- 관리자 현황에서 직원별 기록 수정 후 개인 화면에도 반영된다.
