# 프로젝트 관리

## 기본 정보

- Route: `/project-management`
- 주요 파일: `src/pages/ProjectManagementPage.tsx`
- 주요 사용자: 직원, 관리자, 프로젝트 담당자
- 화면 목적: 클라이언트/내부 프로젝트를 검색하고 상태, 입금, 견적, 담당자를 관리한다.

## UI 구조

- 좌측/목록: 프로젝트 검색, 클라이언트/내부 구분, 프로젝트 카드
- 상세: 프로젝트명, 견적 출처, 개요, 타임라인, 수익성, 마일스톤
- 사이드 패널: 단계, 입금 상태, 예상 견적, 생성일, 담당 직원
- 액션: 수정, 삭제, 담당자 배정, 상태 변경

## 톤앤매너

- 프로젝트명과 현재 단계가 가장 먼저 읽혀야 한다.
- 완료/입금/진행 예정 같은 상태는 정확한 의미로 표시한다.
- 긴 프로젝트명은 줄바꿈 또는 truncate 기준을 통일한다.

## 자주 수정하는 요소

- 프로젝트 카드 안의 정보 순서
- 단계/입금 상태 배지
- 상세 탭 구성
- 담당자 배정 UI

## 연결 데이터와 주의점

- 연결 데이터: `projects`, `saved_quotes`, `project_assignments`, `project_milestones`
- 권한: 담당자와 관리자 권한 분리
- 변경 위험: 입금 상태를 완료 상태로 오해하게 만드는 UI 표현을 피한다.

## 수정 체크리스트

- [ ] 완료 상태와 입금 상태가 혼동되지 않는가?
- [ ] 견적서에서 생성된 프로젝트임을 명확히 표시하는가?
- [ ] 프로젝트 목록과 상세의 상태 표현이 일치하는가?
- [ ] 삭제 버튼은 실수 방지 UI를 유지하는가?
