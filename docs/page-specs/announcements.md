# 공지사항

## 기본 정보

- Route: `/announcements`
- 주요 파일: `src/pages/AnnouncementsPage.tsx`, `src/components/AnnouncementCard.tsx`
- 주요 사용자: 전체 직원, 관리자
- 화면 목적: 회사 공지, 일정, 이벤트, 업무 안내를 작성하고 확인한다.

## UI 구조

- 상단: 공지사항 제목, 작성 액션, 필터/검색
- 목록: 공지 유형, 제목, 날짜, 작성자, 요약
- 상세/작성: 제목, 내용, 일정, 공개 범위
- 보조: 빈 상태, 미팅/이벤트 배지

## 톤앤매너

- 공지는 업무 안내이므로 제목과 날짜가 명확해야 한다.
- 유형 배지는 과하게 튀지 않게 작고 일관되게 사용한다.
- 홈 카드에서는 최근 공지만 간결하게 보여준다.

## 자주 수정하는 요소

- 공지 카드 제목/날짜 레이아웃
- 유형 배지
- 작성 버튼과 전체보기 액션
- 빈 상태 문구

## 연결 데이터와 주의점

- 연결 데이터: 공지/일정 관련 테이블, 홈 대시보드 카드
- 권한: 작성/수정은 관리자 또는 허용된 역할
- 변경 위험: 일정형 공지는 캘린더/홈 카드와 연결될 수 있다.

## 정보 위계

- 1순위: 공지 제목, 날짜/시간, 유형
- 2순위: 작성자, 요약, 관련 일정
- 3순위: 상세 내용, 첨부/링크
- 낮은 강조: 오래된 공지, 보조 메타

## 상세 설계 기준

- 목록 카드: 유형 배지, 제목, 날짜를 같은 순서로 두고 제목은 두 줄까지 허용한다.
- 작성 액션: 작성 버튼은 관리자/허용 역할에게만 명확히 보이게 한다.
- 홈 카드: 최근 공지 3건 정도만 보여주고 전체보기 액션은 헤더 우측에 둔다.
- 일정형 공지: 날짜 범위와 시간은 캘린더 카드와 같은 형식으로 표시한다.
- 모바일: 공지 카드 높이가 과도하게 길어지지 않도록 요약은 최대 2-3줄로 제한한다.

## 검수 시나리오

- 미팅/이벤트/일반 공지가 각 유형 배지를 올바르게 표시한다.
- 홈 공지 카드와 전체 공지 목록의 날짜/제목이 일치한다.
- 작성 권한이 없는 계정에서 작성 버튼이 보이지 않거나 차단된다.
- 긴 제목과 긴 요약이 카드 간격을 깨지 않는다.
