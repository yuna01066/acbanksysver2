# 원판 발주 관리

## 기본 정보

- Route: `/material-orders`
- 주요 파일: `src/pages/MaterialOrdersPage.tsx`
- 주요 사용자: 발주 담당 직원, 관리자
- 화면 목적: 프로젝트/견적에 필요한 원판 발주 내역과 상태를 관리한다.

## UI 구조

- 상단: 원판 발주 관리 제목, 신규 발주/필터 액션
- 목록: 발주명, 거래처/프로젝트, 소재/사이즈/수량, 상태, 일정
- 상세: 발주 품목, 연결 프로젝트, 메모, 진행 상태
- 보조: 상태별 필터와 검색

## 톤앤매너

- 발주 상태와 품목 정보가 빠르게 스캔되어야 한다.
- 원판 사이즈/두께/색상은 같은 표기 규칙을 따른다.
- 상태 변경 버튼은 다른 일반 버튼보다 의미가 분명해야 한다.

## 자주 수정하는 요소

- 발주 카드/테이블의 품목 표시
- 상태 배지와 필터
- 프로젝트 연결 표시
- 빈 상태 문구

## 연결 데이터와 주의점

- 연결 데이터: `material_orders`, `projects`, `saved_quotes`
- 권한: 발주 수정은 관리자/담당자 권한 확인
- 변경 위험: 발주 수량과 견적 수량을 임의 동기화하지 않는다.

- [ ] 프로젝트 연결이 없을 때도 화면이 깨지지 않는가?
- [ ] 모바일에서 긴 발주명이 버튼과 겹치지 않는가?
