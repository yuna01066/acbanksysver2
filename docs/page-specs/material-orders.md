# 원판 발주 관리

## 기본 정보

- Route: `/material-orders`
- 주요 파일: `src/pages/MaterialOrdersPage.tsx`
- 주요 사용자: 발주 담당 직원, 관리자
- 화면 목적: 프로젝트/견적에 필요한 원판 발주 내역과 상태를 관리한다.

## UI 구조

- 상단: 원판 발주 관리 제목, 신규 발주/필터 액션
- 목록: 발주명, 거래처/프로젝트, 소재/사이즈/수량, 상태, 일정
- 상세: 발주 품목, 연결 프로젝트, 메모, 진행 상태
- 보조: 상태별 필터와 검색

## 톤앤매너

- 발주 상태와 품목 정보가 빠르게 스캔되어야 한다.
- 원판 사이즈/두께/색상은 같은 표기 규칙을 따른다.
- 상태 변경 버튼은 다른 일반 버튼보다 의미가 분명해야 한다.

## 자주 수정하는 요소

- 발주 카드/테이블의 품목 표시
- 상태 배지와 필터
- 프로젝트 연결 표시
- 빈 상태 문구

## 연결 데이터와 주의점

- 연결 데이터: `material_orders`, `projects`, `saved_quotes`
- 권한: 발주 수정은 관리자/담당자 권한 확인
- 변경 위험: 발주 수량과 견적 수량을 임의 동기화하지 않는다.

## 정보 위계

- 1순위: 발주 상태, 품목/규격, 수량, 연결 프로젝트
- 2순위: 발주일/입고 예정일, 담당자, 거래처
- 3순위: 메모, 첨부 문서, 내부 참조
- 낮은 강조: 오래된 상태 변경 기록

## 상세 설계 기준

- 규격 표기: 소재, 컬러, 두께, 사이즈, 수량을 같은 순서로 표기한다.
- 상태 배지: 요청/발주/입고/취소 같은 단계는 색보다 문구로 먼저 구분한다.
- 목록 밀도: 발주는 반복 확인 화면이므로 카드형보다 테이블형 밀도를 우선한다.
- 연결 정보: 프로젝트가 있으면 프로젝트명, 없으면 "미연결"을 명확히 표시한다.
- 모바일: 품목 규격은 줄바꿈을 허용하고 상태/수량은 상단에 고정적으로 보이게 한다.

## 검수 시나리오

- 프로젝트 연결 발주와 미연결 발주가 모두 안정적으로 표시된다.
- 입고 상태 변경 후 필터와 요약이 갱신된다.
- 긴 규격명과 다중 품목이 목록 폭을 깨지 않는다.
- 삭제/취소 액션이 즉시 실행되지 않고 확인 단계를 갖는다.
