# 상담 응대 관리

## 기본 정보

- Route: `/response-assistant-management`
- 주요 파일: `src/pages/ResponseAssistantManagementPage.tsx`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 상담 응대 보조에 쓰이는 지침, 근거, 템플릿을 관리한다.

## UI 구조

- 상단: 관리 제목, 새 지침/저장 액션
- 목록: 지침/템플릿 제목, 카테고리, 상태, 수정일
- 편집: 제목, 본문, 태그, 사용 여부
- 보조: 미리보기, 검증 메시지

## 톤앤매너

- 지침 편집 화면은 문서 편집기처럼 차분하고 넓게 유지한다.
- 활성/비활성 상태를 명확히 보여준다.
- 긴 템플릿은 접힘 또는 미리보기로 관리한다.

## 자주 수정하는 요소

- 카테고리/태그 표시
- 편집 폼 라벨
- 미리보기 영역
- 저장/비활성 액션

## 연결 데이터와 주의점

- 연결 데이터: 응대 지침/템플릿 테이블, AI 응대 생성 흐름
- 권한: 관리자/중간관리자 전용
- 변경 위험: 잘못된 지침은 상담 답변 품질에 직접 영향을 준다.

## 정보 위계

- 1순위: 활성 지침, 카테고리, 지침 본문
- 2순위: 태그, 수정일, 작성자, 미리보기
- 3순위: 비활성 지침, 내부 설명
- 낮은 강조: 과거 버전 로그

## 상세 설계 기준

- 편집 구조: 목록과 편집 패널을 분리해 현재 편집 중인 지침이 명확히 보이게 한다.
- 미리보기: 상담원이 보게 될 추천 답변 형태를 저장 전 확인할 수 있게 한다.
- 상태: 활성/비활성은 배지와 토글을 함께 사용하되 삭제보다 비활성을 우선한다.
- 태그: 샘플칩, 제작, 견적, 도면, 납기 등 실제 문의 분류와 연결되는 태그를 사용한다.
- 모바일: 긴 본문 편집은 전체 폭을 사용하고 목록은 접힌 선택 UI로 전환한다.

## 검수 시나리오

- 지침을 비활성화하면 응대 보조 결과에서 제외된다.
- 저장 전 미리보기와 실제 응대 보조 화면 결과 문구가 크게 어긋나지 않는다.
- 카테고리/태그 검색으로 필요한 지침을 찾을 수 있다.
- 일반 직원이 management route에 직접 접근해도 차단된다.
