# 상담 응대 보조

## 기본 정보

- Route: `/response-assistant`
- 주요 파일: `src/pages/ResponseAssistantPage.tsx`
- 주요 사용자: 상담 직원, 관리자
- 화면 목적: 고객 문의에 대한 내부 응대 문안과 근거를 빠르게 작성한다.

## UI 구조

- 상단: 로고/홈 이동, 검색 또는 입력 영역, 관리 화면 이동
- 본문: 문의 입력, 추천 답변, 근거, 복사 액션
- 보조: 관련 태그, 참고 문서, 빈 상태
- 오류 상태: 분석 실패 또는 근거 없음 안내

## 톤앤매너

- 상담원이 빠르게 읽고 복사할 수 있게 문장과 버튼을 간결하게 유지한다.
- 추천 답변은 내부 참고용이며 최종 발송은 사람이 확인한다는 톤을 유지한다.
- 입력 영역과 결과 영역의 시각적 위계를 명확히 둔다.

## 자주 수정하는 요소

- 입력 placeholder
- 추천 답변 카드
- 복사 버튼과 성공 메시지
- 근거 없음 안내

## 연결 데이터와 주의점

- 연결 데이터: 응대 지침/근거 데이터, AI Gateway 또는 Edge Function
- 권한: 로그인 사용자 중심, 관리 기능은 관리자 권한
- 변경 위험: 고객에게 직접 발송하는 흐름으로 바꾸지 않는다.

## 정보 위계

- 1순위: 문의 입력, 추천 답변, 복사 액션
- 2순위: 답변 근거, 관련 태그, 확인 필요 항목
- 3순위: 관리 화면 이동, 히스토리/보조 정보
- 낮은 강조: AI 내부 처리 문구

## 상세 설계 기준

- 입력: 상담원이 그대로 붙여넣기 쉬운 큰 입력 영역을 유지하고 placeholder는 사용 예시 중심으로 작성한다.
- 결과: 추천 답변, 확인 체크리스트, 근거를 카드로 분리해 답변과 판단 근거가 섞이지 않게 한다.
- 복사: 답변 복사 버튼은 가장 명확한 primary 액션으로 두고 복사 성공 toast를 제공한다.
- 오류: 분석 실패는 재시도 가능하게 안내하고 빈 결과처럼 보이지 않게 한다.
- 모바일: 입력과 결과가 세로로 쌓이며 복사 버튼은 결과 카드 하단에 유지한다.

## 검수 시나리오

- 짧은 문의와 긴 문의 모두 입력/결과 카드가 깨지지 않는다.
- 추천 답변이 없을 때 근거 없음/재시도 안내가 나온다.
- 복사 버튼을 눌렀을 때 클립보드 동작과 피드백이 확인된다.
- 관리자 설정 이동은 권한 없는 사용자에게 혼동을 주지 않는다.
