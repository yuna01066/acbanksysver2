# 구성원 관리

## 기본 정보

- Route: `/employee-profiles`
- 주요 파일: `src/pages/EmployeeProfileManagementPage.tsx`
- 주요 사용자: 관리자, 중간관리자, 직원
- 화면 목적: 직원 프로필, 인사 정보, 권한, 문서함, 근로 관련 정보를 관리한다.

## UI 구조

- 상단: 구성원 관리 제목, 탭, 빠른 이동
- 좌측/목록: 검색, 필터, 구성원 리스트
- 상세: 프로필, 인사 정보, 근무/휴가, 급여/계약, 문서함, 권한
- 액션: 수정, 담당자 배정, 문서 업로드, 권한 변경

## 톤앤매너

- 인사 정보는 밀도 있게 보여주되 섹션 구분이 명확해야 한다.
- 빠른 이동이 탭이나 버튼을 가리지 않도록 상단 여백을 확보한다.
- 사진/이름/부서/연락처는 정렬 기준을 통일한다.

## 자주 수정하는 요소

- 구성원 카드 정보 순서
- 상세 탭 레이블
- 권한 설정 영역
- 빠른 이동과 상단 탭 간격

## 연결 데이터와 주의점

- 연결 데이터: `profiles`, `user_roles`, 근태/연차/문서 관련 테이블
- 권한: 개인정보와 권한 변경은 관리자 중심
- 변경 위험: 개인정보가 많은 화면이므로 테스트/문서에 실제 민감 정보를 남기지 않는다.

## 수정 체크리스트

- [ ] 빠른 이동이 상단 탭을 가리지 않는가?
- [ ] 일반 직원에게 민감 정보/권한 변경 액션이 보이지 않는가?
- [ ] 프로필 목록과 상세 선택 상태가 일치하는가?
- [ ] 모바일에서 상세 탭이 가로 스크롤 가능하게 유지되는가?
