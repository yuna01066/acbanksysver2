# 구성원 관리

## 기본 정보

- Route: `/employee-profiles`
- 주요 파일: `src/pages/EmployeeProfileManagementPage.tsx`
- 주요 사용자: 관리자, 중간관리자, 직원
- 화면 목적: 직원 프로필, 인사 정보, 권한, 문서함, 근로 관련 정보를 관리한다.

## UI 구조

- 상단: 구성원 관리 제목, 탭, 빠른 이동
- 좌측/목록: 검색, 필터, 구성원 리스트
- 상세: 프로필, 인사 정보, 근무/휴가, 급여/계약, 문서함, 권한
- 액션: 수정, 담당자 배정, 문서 업로드, 권한 변경

## 톤앤매너

- 인사 정보는 밀도 있게 보여주되 섹션 구분이 명확해야 한다.
- 빠른 이동이 탭이나 버튼을 가리지 않도록 상단 여백을 확보한다.
- 사진/이름/부서/연락처는 정렬 기준을 통일한다.

## 자주 수정하는 요소

- 구성원 카드 정보 순서
- 상세 탭 레이블
- 권한 설정 영역
- 빠른 이동과 상단 탭 간격

## 연결 데이터와 주의점

- 연결 데이터: `profiles`, `user_roles`, 근태/연차/문서 관련 테이블
- 권한: 개인정보와 권한 변경은 관리자 중심
- 변경 위험: 개인정보가 많은 화면이므로 테스트/문서에 실제 민감 정보를 남기지 않는다.

## 정보 위계

- 1순위: 직원명, 재직 상태, 역할/권한, 연락처
- 2순위: 부서, 입사일, 근태/연차 요약, 문서 상태
- 3순위: 급여/계약, 전자계약, 기타 정보
- 낮은 강조: 내부 처리 메모, 오래된 변경 기록

## 상세 설계 기준

- 좌측 목록: 사진/이름/상태를 같은 정렬로 맞추고 선택된 직원은 배경보다 border로 차분하게 강조한다.
- 상세 헤더: 이름, 역할, 재직 기간, 연락처를 한 덩어리로 읽히게 한다.
- 탭: 인사 정보, 근무/휴가, 급여/계약, 문서함, 권한은 같은 높이와 간격을 유지한다.
- 빠른 이동: fixed quick nav와 상세 탭이 겹치지 않게 페이지 상단 여백을 둔다.
- 모바일: 직원 목록과 상세를 분리하고 민감 정보 섹션은 접힘 상태를 기본으로 고려한다.

## 검수 시나리오

- 직원 선택 시 상세 정보가 즉시 바뀌고 이전 직원 정보가 남지 않는다.
- 중간관리자 계정에서 권한 변경 가능 범위가 정책과 맞는지 확인한다.
- 빠른 이동 버튼이 탭/권한 버튼을 가리지 않는다.
- 프로필 사진이 없을 때 fallback 아바타가 균일하게 표시된다.
