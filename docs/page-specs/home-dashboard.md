# 홈 대시보드

## 기본 정보

- Route: `/`
- 주요 파일: `src/pages/Home.tsx`
- 주요 사용자: 전체 직원, 관리자, 중간관리자
- 화면 목적: 오늘 처리할 일, 채널톡 문의, 공지, 프로젝트, 출근/채팅/빠른 이동을 한 화면에서 확인한다.

## UI 구조

- 상단: 로고/테마 전환, 알림, 팀/마이페이지/로그아웃, 빠른 이동
- 핵심 카드: 인사/근무 상태, 오늘 처리할 일, 채널톡 문의, 견적/출근 관련 카드
- 보조 카드: 공지사항, 팀 활동, 프로젝트 진행률, 현재 출근 중, 팀 채팅
- 링크 영역: 업무, 견적/프로젝트, 관리, 외부 링크 그룹

## 톤앤매너

- 기존 glass/metal 분위기는 유지하되 홈은 가장 정돈된 기준 화면으로 관리한다.
- 업무 우선순위가 장식보다 먼저 보여야 한다.
- 카드 헤더의 아이콘, 제목 크기, 카운트 배지는 같은 규칙을 따른다.

## 자주 수정하는 요소

- 인사 문구와 근무 상태 요약
- 오늘 처리할 일/채널톡 문의 카드 높이와 스크롤
- 빠른 링크 그룹과 관리자 전용 배지
- 공지/프로젝트/팀 채팅 빈 상태 문구

## 연결 데이터와 주의점

- 연결 데이터: `notifications`, `projects`, `saved_quotes`, `channel_talk_quote_leads`, 근태/채팅 관련 테이블
- 권한: 로그인 사용자가 기본이며 관리자 전용 링크는 역할 체크가 필요하다.
- 변경 위험: 홈 카드는 여러 도메인 데이터를 동시에 읽으므로 쿼리 비용과 로딩 상태를 신경 써야 한다.

## 정보 위계

- 1순위: 오늘 처리할 일, 채널톡 문의, 출근/퇴근 상태
- 2순위: 공지사항, 프로젝트 진행률, 견적 관련 요약
- 3순위: 팀 채팅, 팀 활동, 현재 출근 중
- 낮은 강조: 명언, 장식성 배너, 외부 링크

## 상세 설계 기준

- 레이아웃: 상단 quick nav와 페이지 내용 사이에 충분한 여백을 두고, fixed/float 요소가 버튼을 가리지 않게 한다.
- 카드: 홈 카드 헤더는 아이콘 20px 내외, 제목 16-18px, 카운트 배지는 우측 정렬을 기본으로 한다.
- 스크롤: 오늘 처리할 일과 채널톡 문의는 같은 높이 기준을 쓰고, 항목이 많으면 카드 내부만 스크롤한다.
- 링크 그리드: 업무/견적/관리/외부 링크로 묶고 빈 placeholder 카드는 만들지 않는다.
- 모바일: 핵심 카드가 한 열로 쌓이며 상단 버튼은 아이콘 버튼 중심으로 줄인다.

## 검수 시나리오

- 로그인 직후 홈 진입 시 첫 화면에서 오늘 처리할 일과 채널톡 문의가 동시에 보인다.
- 데이터가 없을 때 공지, 채팅, 프로젝트 카드가 빈 상태 문구를 같은 톤으로 보여준다.
- 일반 직원 계정에서 관리자 링크가 직접 실행되지 않는다.
- 모바일 폭에서 빠른 이동 버튼이 헤더나 탭을 가리지 않는다.
