# 견적 초안함

## 기본 정보

- Route: `/quote-drafts`
- 주요 파일: `src/pages/QuoteDraftsPage.tsx`, `src/services/quoteDrafts.ts`, `src/contexts/QuoteContext.tsx`
- 주요 사용자: 견적 작성 직원
- 화면 목적: 여러 견적 초안을 저장, 전환, 복제, 보관하고 필요한 초안을 발행한다.

## UI 구조

- 상단: 초안함 제목, 새 초안/계산기 이동 액션
- 목록: 초안명, 수정일, 수신자/합계/상태, 열기/복제/보관/발행 액션
- 선택 발행: 체크박스와 발행 결과 피드백
- 빈 상태: 초안이 없을 때 새 초안 생성 유도

## 톤앤매너

- 브라우저 탭처럼 빠르게 전환한다는 느낌을 유지한다.
- 초안명과 마지막 수정 시간이 먼저 읽혀야 한다.
- 발행 견적서 목록과 혼동되지 않도록 초안 상태 배지를 분명히 둔다.

## 자주 수정하는 요소

- 초안 카드/테이블 레이아웃
- 저장 상태/발행 상태 배지
- 선택 발행 결과 메시지
- 빈 상태 안내

## 연결 데이터와 주의점

- 연결 데이터: `quote_drafts`, `saved_quotes`
- 권한: 작성자 본인 초안만 조회/수정/삭제
- 변경 위험: 초안은 실제 견적번호를 확정하지 않고 발행 시점에 확정한다.

## 정보 위계

- 1순위: 초안 제목, 최근 수정일, 총액, 발행 가능 여부
- 2순위: 수신자/프로젝트 요약, 항목 수, 상태 배지
- 3순위: 복제, 보관, 삭제 같은 보조 액션
- 낮은 강조: 내부 ID, 자동 저장 세부 로그

## 상세 설계 기준

- 목록: 초안명은 한 줄 우선, 길면 두 줄까지 허용하고 날짜/금액은 우측 또는 보조 열에 둔다.
- 탭형 전환: 계산기 내부 툴바는 브라우저 탭처럼 현재 초안만 강조하고 나머지 액션은 compact하게 둔다.
- 선택 발행: 체크박스 선택, 발행 가능/불가 사유, 결과 피드백을 같은 영역에서 보여준다.
- 보관/삭제: 삭제보다 보관을 기본 액션처럼 보이게 하고 파괴적 액션은 낮은 빈도로 배치한다.
- 모바일: 카드형 목록으로 전환하고 주요 액션은 더보기 메뉴나 하단 액션 행에 둔다.

## 검수 시나리오

- 초안 2개를 만들어 전환했을 때 수신자와 항목이 섞이지 않는다.
- 필수값 누락 초안은 선택 발행에서 실패 사유가 보인다.
- 발행 완료 초안은 발행 견적서 목록에 새 견적번호로 나타난다.
- 보관한 초안은 기본 active 목록에서 사라지고 복구 정책이 문구로 명확하다.
