# 발행 견적서 목록

## 기본 정보

- Route: `/saved-quotes`
- 주요 파일: `src/pages/SavedQuotesPage.tsx`
- 주요 사용자: 견적 작성 직원, 관리자
- 화면 목적: 발행된 견적서를 검색, 필터링, 프로젝트 연결, 상태 관리한다.

## UI 구조

- 상단: 제목, 견적서 작성, 공간디자인 견적서, 홈/로고 이동
- 요약 카드: 표시 견적 수, 합계, 프로젝트 연결 현황
- 검색/필터: 검색어, 날짜, 담당자, 단계, 정렬
- 목록: 발행일, 견적 제목, 거래처, 프로젝트, 단계, 금액, 작업 액션

## 톤앤매너

- 견적번호보다 견적 제목과 발행일이 먼저 읽히게 한다.
- 테이블은 업무용 밀도를 유지하되 행 간격과 텍스트 위계를 통일한다.
- 상태 배지는 작고 명확하게 유지한다.

## 자주 수정하는 요소

- 행 안의 텍스트 순서
- 발행일/견적 제목/견적번호 강조 순서
- 필터 칩과 초기화 버튼
- 프로젝트 생성/연결 버튼

## 연결 데이터와 주의점

- 연결 데이터: `saved_quotes`, `projects`, 사용자 프로필
- 권한: 관리자는 전체 담당자 조회, 일반 사용자는 본인/허용 범위 기준
- 변경 위험: 유효기간 만료 자동 상태 변경, 프로젝트 연결 생성, 삭제 액션과 연결된다.

## 정보 위계

- 1순위: 발행일, 견적 제목, 거래처, 총액
- 2순위: 견적번호, 프로젝트 연결 상태, 현재 단계
- 3순위: 담당자, 작성자, 복제/삭제/상세 액션
- 낮은 강조: 내부 메모성 보조 정보

## 상세 설계 기준

- 목록 배열: 발행일을 첫 열 또는 행 첫 정보로 두고 견적 제목은 견적번호보다 굵게 표시한다.
- 행 높이: 한 행에서 제목/거래처/프로젝트/상태/금액/작업이 균형 있게 보이도록 과한 카드 장식을 피한다.
- 필터: 검색, 날짜, 담당자, 단계, 정렬은 한 줄 우선이며 좁은 화면에서는 2줄로 자연스럽게 접는다.
- 액션: 상세 보기, 복제, 삭제 아이콘은 같은 크기로 맞추고 삭제는 색상으로만 과하게 강조하지 않는다.
- 관리자 표시: 관리자에게만 전체 담당자 조회 문구를 보이고 일반 직원에게는 불필요한 권한 문구를 숨긴다.

## 검수 시나리오

- 최근 날짜순 정렬에서 발행일이 가장 먼저 읽히는지 확인한다.
- 제목이 없는 견적은 안전한 fallback 제목으로 보인다.
- 프로젝트 미연결 견적에서 프로젝트 생성 버튼이 동작하고 목록 상태가 갱신된다.
- 필터 초기화 후 요약 카드 수치와 목록 수가 일치한다.
