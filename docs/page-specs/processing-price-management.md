# 가공 가격 관리

## 기본 정보

- Route: `/processing-price-management`
- 주요 파일: `src/pages/ProcessingPriceManagement.tsx`
- 주요 사용자: 관리자, 중간관리자
- 화면 목적: 재단, 접착, CNC, 미러, 대량 제작 등 가공 옵션과 계산 기준을 관리한다.

## UI 구조

- 상단: 가공 가격 관리 제목, 카테고리/저장 액션
- 본문: 가공 옵션 목록, 계산 방식, 단가/배수/고정비, 검수 필요 여부
- 보조: 생산 불가 조건, 안내 문구, 오류 표시
- 액션: 옵션 추가/수정/비활성

## 톤앤매너

- 자유 수식처럼 보이지 않게 검증된 계산 방식 중심으로 보여준다.
- 접착가공은 일반 접착, 무기포 45도, 무기포 90도 등 의미를 명확히 나눈다.
- 관리자 입력 실수를 줄이기 위해 라벨과 도움말을 짧게 둔다.

## 자주 수정하는 요소

- 계산 방식 라벨
- 옵션 설명/도움말
- 검수 필요/생산 불가 표시
- 접착 관련 입력 항목

## 연결 데이터와 주의점

- 연결 데이터: `processing_options`, `advanced_processing_settings`, 가격 계산 엔진
- 권한: 관리자/중간관리자 전용
- 변경 위험: DB 옵션과 하드코딩 계산 로직이 중복 적용되지 않도록 주의한다.

## 정보 위계

- 1순위: 가공명, 계산 방식, 적용 단가/배수, 사용 여부
- 2순위: 적용 조건, 검수 필요 여부, 생산 불가 조건
- 3순위: 설명, 내부 운영 메모
- 낮은 강조: 사용하지 않는 레거시 옵션

## 상세 설계 기준

- 계산 방식: 고정비, 원판금액 배수, 원판금액 추가율, m당 단가, 개당 단가, 코너당 단가를 구분해서 표시한다.
- 접착: 원판 기준과 제품 제작 기준의 접착 계산을 같은 이름으로 섞지 않는다.
- 검수 표시: `계산 가능`, `검수 필요`, `생산 불가`를 서로 다른 상태로 관리한다.
- 입력 제한: 자유 수식 입력처럼 보이는 UI는 피하고 검증된 방식 선택 + 숫자 입력 구조를 사용한다.
- 모바일: 관리자 편집은 접힌 카드로 전환하고 위험한 저장 버튼은 화면 하단 고정으로 두지 않는다.

## 검수 시나리오

- 무기포 45도/90도 선택 시 계산 근거가 한 번만 적용된다.
- 5T 대형 박스처럼 제작 불가 조건이 금액 계산보다 먼저 차단된다.
- 비활성 가공 옵션은 계산기 선택지에 나오지 않는다.
- 저장 후 계산기에서 같은 라벨과 상태로 반영된다.
