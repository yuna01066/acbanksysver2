# 발행 견적서 상세

## 기본 정보

- Route: `/saved-quotes/:id`
- 주요 파일: `src/pages/SavedQuoteDetailPage.tsx`, `src/components/QuoteSummaryHeader.tsx`
- 주요 사용자: 견적 작성 직원, 관리자
- 화면 목적: 저장된 견적의 상세, 인쇄/PDF, 프로젝트 연결, 메모, 계산 근거를 확인한다.

## UI 구조

- 상단: 로고 홈, 목록 이동, 수정, 고객용/내부용 PDF 액션
- 견적 본문: 회사/수신자 정보, 견적 항목, 합계, 안내 문구
- 보조 패널: 연결 프로젝트, 메모, 파일/첨부, 계산 근거
- 인쇄 영역: A4 기준 출력 밀도와 파일명 규칙

## 톤앤매너

- 화면용 액션과 인쇄용 견적서 영역을 명확히 분리한다.
- 고객에게 보이는 문구는 내부 관리 문구보다 차분하고 정돈되게 유지한다.
- 인쇄/PDF 기준에서는 불필요한 조작 UI가 보이지 않아야 한다.

## 자주 수정하는 요소

- 상단 액션 버튼 배열
- 견적 항목의 태그/옵션 표시
- 계산 근거 카드
- PDF/인쇄 스타일

## 연결 데이터와 주의점

- 연결 데이터: `saved_quotes`, `projects`, `document_files`, 첨부 파일
- 권한: 견적 작성자/담당자/관리자 접근 정책 확인 필요
- 변경 위험: 기존 견적 금액은 저장 당시 snapshot 기준으로 보호해야 한다.

## 정보 위계

- 1순위: 견적 제목, 견적번호, 총액, 고객/업체 정보
- 2순위: 견적 항목, 계산 근거, PDF/인쇄 액션
- 3순위: 연결 프로젝트, 메모, 첨부 파일
- 낮은 강조: 내부 처리 로그, 보조 배지

## 상세 설계 기준

- 화면/인쇄 분리: 상단 액션과 사이드 패널은 화면용이며 PDF/인쇄에는 견적서 본문만 안정적으로 남긴다.
- 제목 규칙: 프로젝트명은 `[업체명] 제목` 형태를 우선하고 파일명은 `<아크뱅크 견적서> 견적번호_프로젝트 이름` 규칙을 따른다.
- 항목 태그: 고객에게 불필요한 원판 사이즈/면수 반복 표시는 숨기고, 견적 설명에 필요한 속성만 남긴다.
- 계산 근거: 저장 당시 단가표와 금액 근거를 보여주되 거래처명은 노출하지 않는다.
- 모바일: 액션 버튼은 줄바꿈을 허용하고 PDF 버튼 2개가 같은 시각 체계를 갖게 한다.

## 검수 시나리오

- 고객용 PDF와 내부용 PDF를 각각 열어 내부 정보 노출 여부를 확인한다.
- A4 인쇄 미리보기에서 견적 본문 폭과 푸터가 잘리지 않는다.
- 연결 프로젝트가 없을 때 보조 패널이 빈 상태로 안정적으로 보인다.
- 기존 저장 견적에서 단가 변경 후에도 금액이 변하지 않는다.

## 수정 체크리스트

- [ ] 고객용 PDF에 내부 전용 정보가 노출되지 않는가?
- [ ] A4 인쇄 시 폭 넘침과 페이지 잘림이 없는가?
- [ ] 파일명 규칙이 견적번호와 프로젝트명을 사용해 일관적인가?
- [ ] 계산 근거 표시명이 거래처명을 노출하지 않는가?
