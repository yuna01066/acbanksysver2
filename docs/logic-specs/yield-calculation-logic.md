# 수율/네스팅 계산 로직

## 기본 정보

- 주요 파일: `src/hooks/useYieldCalculator.ts`, `src/utils/yieldOptimization.ts`, `src/utils/panelCombinationCalculator.ts`, `src/utils/optimizedNesting.ts`
- 연결 화면: `/calculator?type=yield`, `/calculator?type=quote`
- 연결 데이터: `panel_masters`, `panel_sizes`, `yield_cut_presets`, `yield_calculation_history`
- 계산 목적: 필요한 조각을 어떤 원판에 배치할지 계산하고, 원판 수량과 재활용 가능한 잔재를 함께 판단한다.

## 입력 흐름

- 사용자는 품질, 두께, 원판 후보, 재단 조각의 가로/세로/수량을 입력한다.
- `useAvailablePanelSizes`는 선택 품질/두께 기준으로 사용 가능한 원판 사이즈와 가격을 가져온다.
- DB `panel_sizes` 활성 데이터가 있으면 실제 치수와 단가를 우선 사용한다.
- DB 데이터가 없으면 정적 단가표와 기본 규격 치수를 fallback으로 사용한다.
- 프리셋은 `yield_cut_presets`, 계산 이력은 `yield_calculation_history`에 저장된다.

## 원판 치수 기준

- DB `actual_width`, `actual_height`가 있으면 최우선 사용한다.
- fallback 기준 치수는 규격명별 기본값을 사용한다.
- 두께별 가용 치수 보정:
  - 1.3T 이상 10T 미만: 기준 치수에서 폭/높이 +20mm
  - 10T 이상 20T 이하: 기준 치수 그대로
  - 20T 초과 30T 이하: 기준 치수에서 폭/높이 -50mm
- 원판 후보는 면적 기준 오름차순으로 정렬한다.

## 네스팅 계산 순서

1. 입력 정규화
   - 조각 가로/세로/수량을 숫자로 변환한다.
   - 수량 0 이하 또는 치수 0 이하 조각은 배치 대상에서 제외한다.
2. 간격 결정
   - 두께 10T 미만은 6mm, 10T 이상은 8mm 간격을 사용한다.
3. 배치 가능성 검사
   - 모든 조각이 회전 포함 원판 안에 들어갈 수 있는지 확인한다.
   - 하나라도 들어갈 수 없으면 `canFitAll=false`, `panelsNeeded=0`으로 반환한다.
4. 전략별 배치
   - `area`, `long-edge`, `wide-first`, `tall-first` 네 가지 정렬 전략을 시도한다.
   - 각 전략은 한 원판씩 채우며 최대 50장까지 반복한다.
   - 후보 좌표는 기존 배치물의 오른쪽/아래 지점에서 생성한다.
   - 회전 배치를 허용하되 회전에는 작은 penalty를 준다.
5. 점수 평가
   - 원판 장수 최소화를 가장 크게 우선한다.
   - 같은 장수에서는 폐기 면적, 조각난 잔재, 재활용 가능 잔재 크기를 함께 본다.
   - 가장 낮은 score의 계획을 최종 추천으로 선택한다.

## 잔재 분석 기준

- 사용 면적은 배치된 조각의 면적 합계다.
- 폐기 면적은 원판 면적에서 사용 면적을 뺀 값이다.
- 재활용 가능 잔재는 최소 300x300mm 이상을 기준으로 한다.
- 후보 잔재:
  - 배치 영역 오른쪽 잔재
  - 배치 영역 아래쪽 잔재
  - 오른쪽 아래 모서리 잔재
- 가장 큰 재활용 가능 사각형을 `largestReusableRect`로 표시한다.
- 재활용 가능성이 낮고 조각난 잔재는 `fragmentationPenalty`로 점수에 불리하게 반영한다.

## 출력 구조

- `piecesPerPanel`: 원판당 평균 배치 조각 수
- `panelsNeeded`: 필요한 원판 장수
- `efficiency`: 전체 원판 면적 대비 조각 면적 비율
- `wasteArea`: 총 잔여 면적
- `canFitAll`: 모든 조각을 배치했는지 여부
- `placedCounts`: 조각 ID별 배치 수량
- `offcut`: 잔재 분석 결과
- `layoutPanels`: 원판별 좌표 배치 결과
- `score`: 추천 우선순위 점수

## 견적 계산 연동

- 수율 계산 결과를 견적 계산으로 넘기면 원판 사이즈/장수/두께/품질이 견적 항목에 반영된다.
- 견적 계산은 수율 결과의 원판 장수와 선택 규격을 기준으로 원장 금액을 계산한다.
- 수율 추천 snapshot은 견적 항목의 계산 근거로 남겨 이후 설명 가능성을 확보한다.
- 수율 결과가 견적 금액을 직접 확정하지 않고, 견적 계산 엔진이 단가/가공/추가금을 다시 계산한다.

## 금지/주의 변경

- 원판 장수 최소화보다 단순 효율만 우선하면 실제 원가가 커질 수 있다.
- 잔재를 무조건 작게 만드는 배치가 항상 좋은 배치가 아니다. 재활용 가능한 큰 사각형 잔재를 보존해야 한다.
- 조각 회전 허용 여부를 바꾸면 견적 결과와 제작 가능성이 크게 달라질 수 있다.
- 두께별 간격 6mm/8mm는 절단 여유와 연결되므로 임의로 줄이면 안 된다.
- `MAX_PANELS` 제한을 늘릴 때는 브라우저 성능과 멈춤 이슈를 같이 검토한다.

## 성능 기준

- 조각 수와 수량이 많을수록 후보 좌표와 전략 반복이 증가한다.
- 대량 계산에서는 입력을 그룹화하거나 같은 치수 조각을 합산해 계산량을 줄이는 것이 우선이다.
- 원판 후보를 전부 계산하기보다 운영상 자주 쓰는 후보를 먼저 계산하고 필요 시 확장하는 방식이 안정적이다.
- 브라우저가 멈추는 수준의 입력은 Web Worker 또는 서버 계산으로 분리하는 것을 검토한다.

## 회귀 테스트 기준

- 원판에 들어갈 수 없는 조각은 `canFitAll=false`와 0장 결과를 반환한다.
- 같은 조각 수에서는 원판 장수가 적은 계획이 우선 선택된다.
- 원판 장수가 같으면 폐기 면적이 적고 재활용 가능 잔재가 큰 계획이 우선 선택된다.
- 두께 10T 미만/이상에서 간격이 각각 6mm/8mm로 적용된다.
- DB actual size가 있을 때 fallback 치수보다 DB 치수를 우선 사용한다.
- 수율 결과를 견적 계산으로 넘겨도 원판 장수와 규격이 변형되지 않는다.
