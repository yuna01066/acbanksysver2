# 견적 계산 입력 흐름과 공식표

이 문서는 현재 코드 기준으로 견적 계산기가 어떤 입력을 받아 어떤 순서로 금액을 계산하는지 공식 중심으로 정리한 문서다. 자세한 구조 설명은 `quote-calculation-logic.md`를 참고한다.

## 기준 용어

- `P`: 원판 기본 단가
- `W`: 원장 금액. 원판 기본 단가에 표면/컬러/조색비를 더한 금액
- `T`: 두께 숫자. 예: `5T` -> `5`
- `Q`: 견적 수량
- `M`: 관리자 설정 또는 DB 옵션의 배수
- `R`: 관리자 설정 또는 DB 옵션의 단가/율
- `B`: 관리자 설정 또는 DB 옵션의 기본 비용
- `L`: 계산용 길이(m). 화면 입력은 mm로 받고 내부 계산에서 `L = 입력값(mm) / 1000`으로 변환한다.
- `C`: 코너 수

## 입력 흐름

1. 사용자가 견적 계산기에서 아래 값을 선택한다.
   - 소재: 현재 자동 계산은 `casting` 기준
   - 품질/재질: CLEAR 유광, 사틴, 아스텔, 보급판 등
   - 두께
   - 원판 사이즈
   - 면수: 단면/양면
   - 컬러/조색비
   - 가공 옵션
   - 접착 옵션
   - 추가 옵션과 수량
2. `usePriceCalculation`이 DB 데이터를 가져온다.
   - 원판 단가: `panel_sizes`
   - CLEAR 기준 단가: `basePanelSizesData`
   - 사이즈별 추가금: `panel_option_surcharges`
   - 두께별 조색비: `color_mixing_costs`
   - 가공 옵션: `processing_options`
   - 고급 가공 설정: `advanced_processing_settings`
3. 단일 원판은 `calculatePrice`를 한 번 호출한다.
4. 여러 원판은 원판별 `W`를 먼저 계산한 뒤 전체 원장 합계 `ΣW`를 기준으로 가공/접착을 한 번 계산한다.
5. 계산 결과는 `totalPrice`, `breakdown`, `status`, `warnings`, `blockedReasons`로 반환된다.

## 계산 순서

1. 소재가 `casting`인지 확인한다.
2. 원판 기본 단가 `P`를 찾는다.
3. 사틴/아스텔, 양단면, 브라이트/진백/스리, 조색비를 더해 `W`를 만든다.
4. 선택한 가공 옵션 금액을 더한다.
5. 선택한 접착 옵션 금액을 더한다.
6. 추가 옵션 금액을 더한다.
7. 생산 불가/검수 필요 조건을 확인한다.
8. 견적 항목으로 추가할 때 100원 단위 반올림과 부가세 계산은 견적 합계 단계에서 처리한다.

## 원판 단가 공식

| 항목 | 현재 공식 |
| --- | --- |
| 원판 기본 단가 | `P = DB panel_sizes.price` 우선, 없으면 정적 단가표 fallback |
| 단가 미등록 | `P <= 0`이면 `blocked` |
| 사틴/아스텔 DB 추가금 있음 | `W = CLEAR 유광 기본가 + satin_astel 추가금` |
| 사틴/아스텔 DB 추가금 없음 | `W = 해당 품질 정적 단가 + 정적 추가금` |
| 양단면 DB 추가금 있음 | `W = W + double_surface 추가금` |
| 양단면 DB 추가금 없음 | `W = W + 정적 양단면 추가금` |
| 브라이트/진백/스리 | `W = W + bright_pigment 추가금` |
| 조색비 | `W = W + color_mixing_costs[두께]` 또는 화면 입력 조색비 |

## 원판 구매/가공 공식

DB `processing_options`에 활성 옵션이 있고 `pricing_method`가 설정되어 있으면 DB 방식이 우선이다. 없거나 값이 비어 있는 알려진 프로필은 코드 기본 배수를 사용한다.

| 항목 | 코드 기본 공식 | 비고 |
| --- | --- | --- |
| 원판 단독 구매 | `추가금 = W × (1.8 - 1)` | 최종 원판 구매 금액은 `W × 1.8`. DB `raw_only_multiplier` 또는 `raw-only` 옵션 배수가 있으면 그 값을 사용 |
| 단순 재단 10T 미만 | `추가금 = W × (1.2 - 1)` | 최종 `W × 1.2` |
| 단순 재단 10T 이상 | `추가금 = W × (1.8 - 1)` | 최종 `W × 1.8` |
| 자동 가공 10T 미만 단순 | `laser-simple` 선택 | 기본 배수 1.7 |
| 자동 가공 10T 미만 복잡 | `laser-complex` 선택 | 기본 배수 2.0 |
| 자동 가공 10T 이상 단순 | `cnc-simple` 선택 | 기본 배수 1.8 |
| 자동 가공 10T 이상 복잡 | `cnc-complex` 선택 | 기본 배수 2.5 |
| 레이저 단순 | `추가금 = W × (1.7 - 1)` | 최종 `W × 1.7` |
| 레이저 복합 | `추가금 = W × (2.0 - 1)` | 최종 `W × 2.0` |
| 레이저 전체 | `추가금 = W × (2.5 - 1)` | 최종 `W × 2.5` |
| CNC 단순 | `추가금 = W × (1.8 - 1)` | 최종 `W × 1.8` |
| CNC 복합 | `추가금 = W × (2.5 - 1)` | 최종 `W × 2.5` |
| CNC 전체 | `추가금 = W × (1.3 - 1) + 300,000` | CNC 전체 재단 정액 공임 300,000원 |
| 엣지 경면 10T 미만 | `추가금 = W × 0.8` | 표시상 최종 `W × 1.8`. 무기포 접착에 엣지가 포함되면 별도 적용하지 않음 |
| 엣지 경면 10T 이상 | `추가금 = W × 0.5` | 표시상 최종 `W × 1.5` |

## 접착 공식

현재 원판 기준 견적의 접착은 단순 배수 방식이다. 문서상 `무기포 = 원판금액 × 3.2`라고 말할 때 실제 코드 구조는 `접착 추가금 = W × (3.2 - 1)`이고, 최종 원판+접착 금액이 `W × 3.2`가 된다.

| 항목 | 현재 공식 | 비고 |
| --- | --- | --- |
| 일반 접착 | `추가금 = W × (2.0 - 1)` | 최종 `W × 2.0` |
| 45도 일반 접착 | `추가금 = W × (2.0 - 1) + 베벨길이 × 베벨단가` | 베벨단가 기본 3,000원/m, DB 설정 우선 |
| 90도 일반 접착 | `추가금 = W × (2.0 - 1) × 1.12 + 코너수 × 코너단가` | 코너단가 기본 4,000원, DB 설정 우선 |
| 무기포 45도, 10T 미만 | `추가금 = W × (3.2 - 1)` | 최종 `W × 3.2`. DB `45-mugipo` 또는 `bond-mugipo-45` 배수가 있으면 우선 |
| 무기포 45도, 10T 이상 | `추가금 = W × (3.3 - 1)` | 최종 `W × 3.3` |
| 무기포 45도 얕은 트레이 | `추가금 = W × (2.0 - 1)` | 제품제작 기준에서 트레이 높이 60mm 이하일 때 |
| 무기포 90도 | `추가금 = W × (3.5 - 1) × 1.12 + 코너수 × 코너단가` | 최종 검수 권장 경고 추가 |
| 무기포 자동 | 45도/90도 계산값 중 낮은 쪽 선택 | 자동 선택 경고 추가 |

## 제품제작 기준 접착 상세 공식

제품제작 기준에서 `useDetailedBond=true`이면 배수 계산에 접착선 길이 기반 상세비가 추가된다.

| 항목 | 공식 |
| --- | --- |
| 수량 할인 계수 | `V(Q) = 1 / (1 + k × ln(max(1, Q)))` |
| 상세 접착비 | `D = (세팅비 / max(1, Q) + m당 접착단가 × 접착선길이) × V(Q) × max(1, Q)` |
| 무기포 45도 제품제작 | `추가금 = W × (45도 배수 - 1) + D + 베벨길이 × 베벨단가` |
| 무기포 90도 제품제작 | `추가금 = W × (3.5 - 1) × 1.12 + 코너수 × 코너단가 + D` |

기본 코드 fallback 값은 세팅비 150,000원, m당 접착단가 42,000원, `k=0.10`이다. 다만 실제 런타임에서는 `advanced_processing_settings`의 활성 값이 우선 적용된다.

## DB 가공 옵션 공식

관리자 패널에서 `processing_options.pricing_method`를 지정하면 아래 공식이 적용된다.

| pricing_method | 공식 |
| --- | --- |
| `fixed_fee` | `금액 = R × Q` |
| `panel_multiplier` | `금액 = W × (M - 1) × Q + B × Q` |
| `panel_rate` | `금액 = W × R × Q + B × Q` |
| `per_unit` | `금액 = R × Q + B × Q` |
| `per_meter` | `금액 = R × L × Q + B × Q` |
| `per_corner` | `금액 = R × C × Q + B × Q` |
| `requires_review` | `금액 = 0`, 검수 필요 사유 추가 |
| `legacy_multiplier` | 별도 방식이 없으면 기존 `multiplier/base_cost` 처리로 fallback |

## 추가 옵션 공식

아래 옵션은 DB 옵션으로 이미 선택된 경우 중복 적용하지 않는다. DB 옵션이 없을 때 코드 기본값을 사용한다.

| 항목 | 기본 공식 |
| --- | --- |
| 엣지 경면 마감 | `금액 = P × 0.5 + B` |
| 불광 마감 | `금액 = P × 0.5 + B` |
| 타공 | `금액 = P × 0.2 + B` |
| 무광 도장 | `금액 = P × 2.0 + B` |
| 레이저 타공 개수 | `금액 = 타공개수 × 타공단가` |

주의: 이 추가 옵션 fallback은 `W`가 아니라 `basePrice(P에 가까운 원판/표면/조색비 계산 전후의 내부 기준값)`를 기준으로 계산되는 경로가 있다. DB `processing_options`로 관리하는 경우는 위 `DB 가공 옵션 공식`을 우선한다.

## 견적 합계 공식

| 항목 | 공식 |
| --- | --- |
| 견적 항목 금액 | `item.totalPrice = 계산 결과 totalPrice` |
| 견적 소계 | `subtotal = Σ(item.totalPrice × item.quantity)` |
| 소계 반올림 | `subtotal = 100원 단위 반올림` |
| 부가세 | `tax = round(subtotal × 0.1)` |
| 총액 | `total = subtotal + tax` 또는 Context 기준 `subtotal × 1.1` 후 100원 단위 반올림 |

## 차단/검수 공식

| 조건 | 결과 |
| --- | --- |
| 원판 단가 없음 | `blocked` |
| 지원하지 않는 소재 | `blocked` |
| 제품제작 기준 박스형 + 5T 이하 + 접착선 길이 9m 이상 | `blocked` |
| 제품제작 기준 박스형 + 5T 이하 + 접착선 길이 7m 이상 | `needs_review` |
| 90도 접착 + 코너 있음 | `needs_review` |
| 무기포 자동 선택 | `needs_review` |
| 제품제작 접착인데 접착선 길이 없음 | `needs_review` |

## 현재 공식 확인 체크리스트

- [ ] 원판 가격은 DB 단가가 있으면 DB 값을 우선 사용한다.
- [ ] 무기포 45도는 추가금이 `W × (배수 - 1)`이고, 최종 원판+접착 기준으로 `W × 배수`가 된다.
- [ ] 원판 기준 접착에는 제품제작용 접착선 길이 입력을 요구하지 않는다.
- [ ] 다중 원판은 각 원판 `W`를 합산한 뒤 가공/접착을 한 번 적용한다.
- [ ] DB `pricing_method`가 있으면 코드 기본 배수보다 DB 공식이 우선한다.
